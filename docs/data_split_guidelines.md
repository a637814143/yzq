# 数据集划分建议

以下策略可用于划分训练集、验证集和测试集，确保模型评估可靠且泛化能力强：

## 常规比例建议
- **训练集：70% ~ 80%** — 用于模型学习和参数拟合。
- **验证集：10% ~ 15%** — 用于调参和早停，避免过拟合。
- **测试集：10% ~ 15%** — 仅在最终评估时使用，提供客观性能估计。

## 数据规模与分布考虑
- **数据量较小（≤1 万样本）**：倾向于 70/15/15 或 75/15/10，保持足够的验证、测试样本以获得稳定估计。
- **数据量较大（>1 万样本）**：可以加大训练占比，如 80/10/10；样本足够时，测试集 5% 亦可提供稳健评估。
- **类别不均衡**：使用分层抽样保持各子集中类别比例一致；必要时在训练集上做重采样或使用 class_weight。

## 特殊场景
- **时间序列或上线监控场景**：按时间顺序划分（先早后晚），避免未来信息泄露；可将最近一段时间留作测试集。
- **跨域或多来源数据**：按来源留出测试集，验证模型跨域泛化能力。

## 交叉验证与早停
- 数据量有限时，可在训练集上使用 K 折交叉验证替代单一验证集，或将验证集融入交叉验证以稳定调参。
- 使用验证集监控指标并在性能不再提升时早停，防止过拟合。

## 实践建议
1. 先按推荐比例划分，确认各子集类别分布接近。
2. 在验证集上完成模型与特征选择，再用测试集一次性报告最终指标。
3. 如需多次实验，固定随机种子或保存划分列表，保证可重复性。
